PropDesk Updates
Latest v1.4.7 May 2026
Trading day boundary is now 6 PM ET everywhere — bot journal calendar, recent activity, and today's trade count all treat post-6PM trades as the next day, matching how prop firm day P&L already worked
Direct Funded profit target bar now uses actual account P&L (allTimePnl) instead of bot journal sum — shows the real balance change toward the target, not just logged trades
Direct Funded payout eligibility now requires BOTH the profit target AND consistency rule to be met — previously only checked consistency
Direct Funded cards now show two progress bars — profit target progress and consistency — so you can see exactly which condition is still outstanding
Added "Correct Peak Balance" field in account config modal — if PropDesk missed your actual highest balance (e.g. account was higher before the app was installed), enter the real peak so trailing drawdown room calculates correctly
Fixed Topstep balance showing ~$25K instead of ~$50K — when TraderPost displays profit-only (e.g. "$1,333"), the server now uses the historically stored peak balance to detect account size correctly instead of guessing from the small profit number
Green day minimum now varies by firm and account size — Lucid 50K requires $150/day, 100K requires $200/day, 150K requires $250/day
Added Direct Funded account type — mark any account as Direct Funded to track the consistency rule (best single day / total cycle profit). Set a custom threshold per account (Lucid Direct: 20%, Tradeify Lightning: varies); a progress bar turns red when you're near the limit and the Near Payout section shows when you're eligible
MLL now uses verified per-firm, per-size dollar amounts (Topstep, Lucid, Apex, Alpha, MFF, FundedNext) instead of a flat 4% estimate — 100K and 150K accounts now display correct room
Fixed MLL room formula when trailing DD has naturally locked — once profit exceeds the loss limit, room correctly shows profit above starting balance instead of an incorrect lower number
Fixed account balances going blank and Today's P&L showing $0 when TraderPost refreshes — server now uses the last known good balance instead of writing nulls when a sync catches the page mid-load
Today's P&L no longer zeroes out after midnight for active after-midnight sessions — day boundary now uses the 6 PM ET cutoff everywhere, not the calendar midnight
Background poll no longer replays P&L events from accounts.json — events only arrive via live sync to prevent double-counting
Payout progress pips are now clickable — click any pip to manually set your green day count; new green days from trades continue to add on top
MLL now accounts for post-payout accounts — account config modal has a toggle to lock MLL at starting balance, auto-enabled when a payout is logged
Fixed account cards flashing blank on every extension sync — cards now build off-screen and swap in one frame
Account cards now show MLL (room left before trailing DD breach) in red next to Peak, which follows the theme color
Trading day now resets at 6 PM ET regardless of system timezone — no timezone configuration needed on any machine
Fixed Prop Accounts firm filter — FILTER label and All button were being wiped on each sync; firm filters now work correctly
Risk Simulator — trailing DD now correctly locks at starting balance once profit exceeds DD limit, matching real prop firm rules
Active accounts number now follows the selected theme color instead of being hardcoded blue
Bot journal now shows one combined entry per account per day — multiple trade closes merge into a single log
